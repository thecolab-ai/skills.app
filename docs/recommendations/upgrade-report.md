## Rejected by verification

The adversarial pass contradicted these rationales — recorded so they aren't
re-proposed:

- **`maxDuration` for run routes** — `/api/run` is fire-and-forget (returns
  `job.id` immediately) and the in-memory `globalThis` run-store is broken on
  serverless regardless. Not a fix.
- **Replace run poll with SSE stream** — real but mis-scoped as a drop-in; the
  run-store only accumulates strings (no per-chunk emitter), so it needs an
  EventEmitter + replay-buffer redesign. Defer.
- **Validate run args against introspected schema** — spawn is already hardened
  (`shell:false`, confined entrypoint); the heuristic regex introspector will
  false-reject valid argparse forms. WARN-only at best. Defer.
- **Convert introspect JSON import to runtime read** — benefits the server
  bundle only (already server-only), and introduces a file-tracing gotcha under
  `output: 'standalone'`. Skip unless build memory is a real problem.
